# 🇺🇸 English

## 🎉 Introduction

**APIDocForge** is a powerful, intelligent API documentation generator that transforms your source code into beautiful, interactive documentation. It automatically extracts API endpoints from your code comments and generates comprehensive documentation in multiple formats.

### ✨ Key Features

- 🔍 **Multi-Language Support** - Parse Python, JavaScript, TypeScript, Go, Java, and Rust code
- 🎨 **Beautiful HTML Docs** - Generate modern, responsive documentation with search
- 📄 **Multiple Formats** - Export to HTML, Markdown, OpenAPI, JSON, and YAML
- 🔄 **Live Reload** - Watch mode for automatic regeneration on file changes
- 🌐 **Built-in Server** - Preview your documentation with the integrated dev server
- 📝 **Smart Parsing** - Extract endpoints from docstrings, JSDoc, and annotations
- 🎯 **OpenAPI Compatible** - Import and export OpenAPI 3.0 specifications

## 🚀 Quick Start

### Installation

```bash
# Install from source
pip install -e .

# Or install with development dependencies
pip install -e ".[dev]"
```

### Basic Usage

```bash
# Generate documentation from current directory
adf generate .

# Generate with specific options
adf generate ./src -o ./docs -t "My API" -v "1.0.0" -f html markdown openapi

# Start preview server
adf preview ./src --port 8080

# Watch for changes
adf watch ./src -o ./docs
```

### Example

Given this Python code:

```python
@app.get("/users/{user_id}")
def get_user(user_id: int):
    """
    Get a specific user by ID.
    
    :param user_id: The user's unique identifier
    :return: User details
    """
    return {"id": user_id, "name": "John"}
```

APIDocForge will automatically generate:
- Interactive HTML documentation
- Markdown API reference
- OpenAPI 3.0 specification
- JSON/YAML exports

## 📖 Detailed Usage

### Command Line Interface

```bash
# Initialize configuration file
adf init

# Generate documentation
adf generate [OPTIONS] [INPUT_PATH]

Options:
  -o, --output PATH       Output directory [default: ./docs]
  -f, --format FORMAT     Output format(s) [default: html]
                          Choices: html, markdown, openapi, json, yaml
  -t, --title TEXT        API title
  -v, --version TEXT      API version
  -d, --description TEXT  API description
  --base-url TEXT         Base URL for API server
  --parser TYPE           Parser type [default: auto]
  --watch                 Watch for file changes
  --serve                 Start development server
  --port INTEGER          Server port [default: 8080]
```

### Configuration File

Create `apidocforge.toml`:

```toml
[tool.apidocforge]
title = "My API"
version = "1.0.0"
description = "API documentation generated by APIDocForge"
input_path = "."
output_path = "./docs"
output_formats = ["html", "openapi"]
parser_type = "auto"
base_url = "https://api.example.com"

[tool.apidocforge.features]
enable_code_samples = true
enable_try_it = true
enable_search = true
```
